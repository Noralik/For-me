### Идея структуры таблиц для персонажей Arknight:

---

#### 1. Таблица `Characters` (основная информация)

| Поле           | Тип      | Описание                      |
| -------------- | -------- | ----------------------------- |
| CharacterID    | INT (PK) | Уникальный ID персонажа       |
| Name           | NVARCHAR | Имя персонажа                 |
| RealName       | NVARCHAR | Настоящее имя                 |
| Age            | INT      | Возраст                       |
| Birthday       | DATE     | День рождения                 |
| Gender         | NVARCHAR | Пол                           |
| Race           | NVARCHAR | Раса                          |
| Height         | FLOAT    | Рост                          |
| ParadoxSimName | NVARCHAR | Название парадокс симуляции   |
| Rarity         | NVARCHAR | Редкость (например, 3 звезды) |

---

#### 2. Таблица `Classes` (классы персонажей)

| ClassID       | INT (PK)    | Уникальный ID класса          |
| ClassName     | NVARCHAR    | Название класса               |

---

#### 3. Таблица `Branches` (ветки специализации)

| BranchID      | INT (PK)    | Уникальный ID ветки           |
| BranchName    | NVARCHAR    | Название ветки                |

---

#### 4. Таблица `Factions` (фракции)

| FactionID     | INT (PK)    | Уникальный ID фракции         |
| FactionName   | NVARCHAR    | Название фракции              |

---

#### 5. Таблица `Positions` (позиции в игре)

| PositionID    | INT (PK)    | Уникальный ID позиции          |
| PositionName  | NVARCHAR    | Название позиции              |

---

#### 6. Таблица `Tags` (теги)

| TagID        | INT (PK)    | Уникальный ID тега             |
| TagName      | NVARCHAR    | Название тега                 |

---

#### 7. Таблица `ObtainingMethods` (как получить персонажа)

| ObtainID     | INT (PK)    | Уникальный ID способа          |
| ObtainName   | NVARCHAR    | Название способа получения    |

---

#### 8. Таблица `CharacterDetails` (связующая, объединяющая таблица)

* Для связи персонажа с классом, фракцией, тегами, способами получения и пр.
* Пример:

| CharacterID (FK) | ClassID (FK) | BranchID (FK) | FactionID (FK) | PositionID (FK) | ObtainID (FK) |
| ---------------- | ------------ | ------------- | -------------- | --------------- | ------------- |

---

#### 9. Таблица `CharacterTags` (многие ко многим, т.к. у персонажа может быть несколько тегов)

| CharacterID (FK) | TagID (FK)  |

---

### Зачем так?

* Если делать всё в одной таблице — будет куча дублирующейся информации.
* Если разбить, то легко поддерживать, обновлять и расширять данные.
* Связи через FOREIGN KEY обеспечивают целостность данных.

---

### Пример запросов создания основных таблиц

```sql
CREATE TABLE Classes (
    ClassID INT PRIMARY KEY IDENTITY(1,1),
    ClassName NVARCHAR(50) NOT NULL
);

CREATE TABLE Characters (
    CharacterID INT PRIMARY KEY IDENTITY(1,1),
    Name NVARCHAR(100),
    RealName NVARCHAR(100),
    Age INT,
    Birthday DATE,
    Gender NVARCHAR(10),
    Race NVARCHAR(50),
    Height FLOAT,
    ParadoxSimName NVARCHAR(100),
    Rarity NVARCHAR(20)
);

CREATE TABLE CharacterClasses (
    CharacterID INT FOREIGN KEY REFERENCES Characters(CharacterID),
    ClassID INT FOREIGN KEY REFERENCES Classes(ClassID),
    PRIMARY KEY (CharacterID, ClassID)
);
```
